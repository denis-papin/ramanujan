# Résumé
`Player` est l'entité racine utilisée pour suivre les buts et les `Award` dérivés.

# Attributs
- id: number, primary key, generated by `player_id_seq`
- name: varchar2(60), nullable in current schema
- goals: number, required, default `0`

# Composants
- award
  - table: award
  - foreign_key: player_id

# Relations
- Player 1..n Award

# Invariants
- `goals >= 0`
- `award_count(player) = floor(goals / 3)`
- `award.issuer in {FIFA, FRANCE_FOOTBALL}`
- `award.award_year in [1980, 2030]`

# Règles métier
- Chaque fois qu'un `Player` gagne un but, l'invariant des `Award` doit être restauré.
- Les `Award` manquants doivent être rattrapés si le total de buts implique plus de récompenses que celles déjà stockées.

# Exemples
- `goals = 0` implique `awards = 0`
- `goals = 3` implique `awards = 1`
- `goals = 25` implique `awards = 8`
