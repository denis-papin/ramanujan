# Résumé
`Player` est l'entité racine utilisée pour suivre les points et les `Award` dérivés.

# Attributs
- id: number, primary key, generated by `player_id_seq`
- name: varchar2(60), nullable in current schema
- points: number, required, default `0`

# Composants
- award
  - table: award
  - foreign_key: player_id

# Relations
- Player 1..n Award

# Invariants
- `points >= 0`
- `award_count(player) = floor(points / 3)`
- `award.issuer in {GOLD, SILVER}`
- `award.award_year in [1980, 2030]`

# Règles métier
- Chaque fois qu'un `Player` gagne un point, l'invariant des `Award` doit être restauré.
- Chaque fois qu'un `Player` perd un point, l'invariant des `Award` doit être restauré.
- Les `Award` manquants doivent être rattrapés si le total de points implique plus de récompenses que celles déjà stockées.
- Les `Award` en trop doivent être supprimés si le total de points implique moins de récompenses que celles déjà stockées.

# Exemples
- `points = 0` implique `awards = 0`
- `points = 3` implique `awards = 1`
- `points = 25` implique `awards = 8`
